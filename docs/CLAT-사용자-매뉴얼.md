## 2. 기능별 소개

### 2.1 계정·로그인 (`/login`, `/signup`)

- **역할:** 선생님 계정으로 서비스에 접속합니다.
- **동작:** 이메일·비밀번호 로그인, 회원가입. 토큰이 만료되면 자동 갱신 후 요청을 재시도하는 구조입니다.
- **내 정보 (`/me`):** 이름·이메일 수정, 비밀번호 변경, 회원 탈퇴 요청(유예 기간 안내 포함).

### 2.2 홈 (`/home`)

- **역할:** 신규 사용자를 위한 **단계별 시작 가이드**(반·학생 등록 → 템플릿 → 수업 입력·발송 → 학생 추적)를 제공합니다.
- 실제 데이터 입력은 각 메뉴에서 진행하고, 홈은 흐름 안내용입니다.

### 2.3 학생·반 관리 (`/management`, `/management/[반ID]`, 학생 상세 `/students/[id]`)

- **반(Class):** 반 생성·수정·종료, 반에 속한 학생 목록 관리.
- **학생:** 개별 등록, **엑셀 일괄 업로드**, 학생 상세·통계·미완료 항목 확인.
- **학생 대시보드:** 해당 학생의 이번 달 완료율·출석률·최근 점수, **점수 추이**·**수업 이력**·**알림톡 이력**, **AI 분석**(선생님용 전용; 아래 §2.9 참고).
- **완료형 항목:** 템플릿에서 “완료/미완료”로 두면, 미완료가 대시보드에 쌓여 **과제·과제형 추적**에 쓰입니다.

### 2.4 수업 템플릿 (`/template`, `/template/new`, `/template/[id]`, `/template/[id]/edit`)

- **역할:** 매 수업마다 반복 입력할 **항목 설계**입니다.
- **공통 항목:** 반 전체가 동일한 내용(예: 오늘 수업 주제).
- **개별 항목:** 학생별로 다른 값.
- **항목 유형(요약):** 텍스트·숫자·선택형·**완료형**(과제 등)·**출결(ATTENDANCE)**·**점수(SCORE)** 등.  
  - v2 정책: **출결(ATTENDANCE) 항목은 템플릿당 정확히 1개만** 허용됩니다. 0개 또는 2개 이상이면 출결 세션 생성 시 API가 거절(422)합니다.
- **문자에 포함:** 항목별로 “알림 문자에 넣을지”를 지정합니다.
- **미리보기:** 템플릿 편집 화면에서 학부모용 문안 형태를 대략 확인할 수 있습니다.

### 2.5 수업 입력 (`/lesson`, `/lesson/new`, `/lesson/[id]`)

- **캘린더:** 날짜를 고르면 그날의 수업 카드가 나옵니다.
- **수업 생성:** 반·템플릿·날짜 선택 후 초안(DRAFT) 작성.
- **입력 화면:** 공통 칸 + 학생별 표. 출결·점수·선택·완료 버튼 등으로 빠르게 입력.
- **저장·완료율:** 저장 후 진행률 표시, 미완료 추적.
- **문자 미리보기 / 엑셀:** 수업 데이터를 바탕으로 학생·학부모 수신 문안 형태 확인, **엑셀 내보내기**(v2 규칙: 상단 학부모 블록 / 빈 행 / 하단 학생 블록, 3열 고정).
- **알림톡 발송:** 학생을 선택해 `POST /lessons/:id/send`로 발송. **학생 번호·학부모 번호 채널별로 각각 1건** 집계됩니다.
- **실시간 출결:** 수업 화면에서 출결 세션 시작·모니터링·종료·수기 수정(플로팅 바·모달). 상세는 §2.7.

### 2.6 알림톡 메뉴 (`/alimtalk`, `/alimtalk/history`)

- **설정:** 인트로·아웃트로 문구 저장. 변수 `{강사명}`, `{학원명}`, `{학생이름}`, `{날짜}`, `{반이름}` 사용.
- **전달 모드 표시:** 서버의 `delivery_mode`가 **mock**(모의)이면 실제 카카오 API 없이 DB·내역만 기록됩니다. **`live`일 때 실연동**입니다.
- **발송 내역:** 배치 목록·상세, 건별 성공/실패·학부모 대시보드용 토큰·AI 피드백 포함 여부 확인.

### 2.7 출결(코드 입력) 흐름

- **선생님:** 수업에 연결된 활성 출결 세션을 만들면 **4자리 코드**와 **만료 시각**이 생깁니다. 학생에게는 **개인별 링크**(세션 ID + `student_id` 쿼리)가 안내됩니다. 알림톡이 성공하면 출결용 배치도 남을 수 있고, **알림 발송이 실패하면 해당 출결용 배치는 만들어지지 않지만 세션은 유지**됩니다(v2 정책).
- **학생:** 공개 URL `/check/[sessionId]?studentId=…`에서 코드 입력. 규칙 요약:
  - 세션 시작 후 **10분 초과** 입력 시 **지각**(단, 세션 제한 시간이 **5분 이하**이면 지각 없이 출석만).
  - 만료·선생님 종료 후에는 입력 불가.
- **선생님 수동 종료 시:** 출결 결과가 수업 레코드의 출결 항목 값에 반영됩니다.
- **개발/테스트용 진입:** `/check`에서 세션 ID·학생 ID를 직접 넣어 출결 화면을 열 수 있습니다.

### 2.8 학부모 공개 대시보드 (`/parent/[토큰]`)

- **역할:** 알림톡(학부모 채널)에 실린 **일회성 링크 토큰**으로, 해당 수업 요약·점수·추가 항목·미완료 목록·최근 수업 목록을 봅니다.
- **링크 유효 시간:** 서버 환경변수 `PARENT_LINK_TTL_HOURS` 기준(저장은 DB `token_expires_at`). 만료 시 **410 TOKEN_EXPIRED**.
- **AI 피드백(학부모용):** 비동기 생성. 화면에서는 **AI가 아직이면 약 5초 간격으로 최대 약 3분** 정도 기다렸다가 안내하는 방식이 가이드입니다.

### 2.9 AI 기능 — 두 가지가 다릅니다 (혼동 금지)

| 구분 | 설정 위치 | 적용 대상 |
|------|-----------|-----------|
| **AI 조교 설정** (`/ai`) | `AiSetting` — 말투·길이·포함 데이터 기간·점수/과제/출결 등 토글 | **알림톡에 붙는 학부모 피드백** 생성 |
| **학생 대시보드 AI 분석** (`/students/[id]`) | 설정 화면과 **공유하지 않음**. 서버 별도 프롬프트·일 캐시 | **선생님이 보는 학생 종합 분석**만 |

즉, `/ai`에서 바꾼 말투가 학생 대시보드 분석 문구에 자동 반영되지는 않습니다.

### 2.10 백엔드가 함께 제공하는 것 (사용자 관점)

- **일관된 REST API**와 Swagger 문서 (`/api-docs`).
- **공개 API 속도 제한**(출결 코드 제출·학부모 페이지 등)으로 남용 완화.
- **재발송 시 학부모 링크:** 같은 수업·같은 학생에 대해 다시 보내면 **이전 학부모 토큰은 즉시 무효화**되고 최신 링크만 유효합니다.

## 3. 화면·메뉴 맵 (선생님 앱 사이드바)

| 메뉴명 | 경로 | 하는 일 |
|--------|------|---------|
| 홈 | `/home` | 시작 가이드 |
| 수업 입력 | `/lesson` … | 캘린더·수업 상세·발송·출결 |
| 학생·반 관리 | `/management` … | 반·학생·엑셀. 학생 상세는 `/students/[id]` |
| 수업 템플릿 | `/template` … | 항목 설계·미리보기 |
| 알림톡 | `/alimtalk` (+ 이력) | 인트로/아웃트로·전달 모드·배치 내역 |
| AI 조교 | `/ai` | 학부모 피드백용 AI 설정·말투 샘플 |
| 내 정보(카드) | `/me` | 프로필·비밀번호·탈퇴 |

**공개(로그인 없음):**

- 학생 출결: `/check/[sessionId]?studentId=…` (테스트 진입 `/check`)
- 학부모: `/parent/[token]`

## 4. 상황별 매뉴얼

### 4.1 서비스를 처음 켰을 때 (온보딩)

1. **회원가입** 후 로그인합니다.
2. **반 관리**에서 반을 하나 만듭니다(이름만으로도 충분히 시작 가능).
3. **반 상세**에서 학생을 추가하거나 **엑셀로 일괄 등록**합니다.  
   - 학부모 전화번호가 있으면 이후 **학부모 알림톡**에 활용됩니다.
4. **수업 템플릿**을 만듭니다.  
   - 출결을 앱으로 할 계획이면 **출결(ATTENDANCE) 항목을 정확히 1개** 넣습니다.
   - 과제 추적이 필요하면 **완료형** 항목을 씁니다.
5. **수업 입력**에서 날짜·반·템플릿으로 수업을 만들고 내용을 채웁니다.
6. **알림톡 설정**에서 인트로/아웃트로를 학원 톤에 맞게 다듬습니다.
7. 필요하면 **AI 조교**에서 학부모 피드백 스타일을 맞춥니다.

**자주 막히는 경우:** 템플릿에 출결 항목이 없거나 두 개 이상이면 **출결 세션 시작이 안 됩니다** — 템플릿을 고칩니다.

---

### 4.2 매일 수업을 기록할 때

1. 사이드바 **수업 입력**으로 이동합니다.
2. 상단에서 **날짜**를 선택합니다.
3. 해당 날짜의 수업 카드에서 **입력하기**로 들어갑니다.
4. **공통 블록**(오늘 진도 등)을 채웁니다.
5. 표에서 학생별로 **출결·점수·선택·과제 완료**를 입력합니다.
6. 작업 중이라면 **저장**을 습관적으로 눌러 두면 안전합니다.
7. 반장·원장에게 공유할 때는 **엑셀 내보내기**를 사용합니다(학부모/학생 블록 구조는 §2.5 참고).

---

### 4.3 학부모·학생에게 알림톡(또는 문자 흐름)을 보낼 때

1. 수업 입력 화면까지 열어 두고, 발송 대상 학생을 선택합니다.
2. **알림톡 발송**을 실행합니다.
3. 결과 토스트·모달에서 **성공/실패 건수**와 **모의(mock) 여부**를 확인합니다.
4. **알림톡 → 발송 내역**에서 배치 단위로 다시 확인할 수 있습니다.

**같은 학생에게 같은 수업으로 다시 보낼 때:**  
이전에 받은 **학부모 링크는 곧바로 무효**됩니다. 학부모에게는 **가장 최근에 받은 링크**만 사용하도록 안내합니다.

**모의 전송(mock)일 때:**  
실제 카카오 도착은 없을 수 있지만, 앱 **내역·DB·토큰**은 연습과 동일하게 쌓입니다. 운영 전환 시 서버 `ALIMTALK_DELIVERY_MODE`가 `live`로 바뀌면 실발송으로 전환됩니다.

---

### 4.4 수업 중 실시간 출결을 할 때 (선생님)

1. 해당 **수업 상세** 화면을 연 상태에서 출결 플로팅 UI를 사용합니다.
2. **출결 시작** 모달에서 **제한 시간(분)**을 고릅니다(프리셋 5·10·15·20분 또는 직접 입력).
3. 시작이 성공하면 **4자리 코드**가 생기고, 학생별 접속 URL이 준비됩니다.  
   - 알림톡으로 링크가 갔다면 학생이 탭하여 코드를 입력합니다.
4. 화면에서 **출석/지각/결석 인원**과 이름별 상태를 실시간으로 봅니다.
5. 필요하면 특정 학생 행을 **수기로 상태 변경**합니다.
6. 수업이 끝나면 **종료**하여 수업 기록의 출결 칸에 결과가 박히게 합니다.

**알림이 실패했다는 로그만 있고 배치가 없을 때:**  
v2 정책상 출결 세션은 살아 있을 수 있으니, **링크/코드를 다른 채널(단톡 등)으로 공유**해 출결을 진행할 수 있습니다.

**이미 한 번 끝낸 출결 세션을 같은 수업에서 “다시 시작”하려 할 때:**  
서버가 **409 SESSION_ALREADY_EXISTS** 등으로 막을 수 있습니다. 그날은 **수기 출결**만으로 보정하거나, 새 수업 레코드 설계가 필요한지 운영 규칙을 정합니다.

---

### 4.5 학생이 출결 코드 화면에서 막혔을 때

| 증상 | 원인 | 조치 |
|------|------|------|
| 코드가 틀리다고 나온다 | 선생님이 공지한 코드와 불일치 | 선생님 화면의 4자리와 대조 |
| 시간이 지났다 | `expires_at` 경과 | 선생님이 세션을 새로 만들거나(가능 시) 수기 반영 요청 |
| 이미 처리됐다 | 중복 제출 | 정상 — 결과 화면 확인 |
| 세션이 종료됐다 | 선생님이 종료 버튼 | 선생님이 수기로 상태 수정 |

**지각 판정:** 시작 후 10분이 지난 코드 입력은 지각입니다(단, 전체 제한 시간이 5분 이하인 수업은 출석만).

---

### 4.6 학부모가 보낸 링크를 열었을 때

1. 카카오 등으로 받은 URL을 엽니다.
2. **만료·잘못된 토큰**이면 에러 문구가 나옵니다 — 선생님께 **재발송**을 요청합니다.
3. 정상이면 해당 수업의 **출결·점수·기타 항목 요약**을 볼 수 있습니다.
4. **AI 코멘트 영역**이 비어 있으면 잠시 후 다시 열어 보세요(생성 대기).  
   - 앱 가이드는 대략 **5초 간격·최대 약 3분** 폴링입니다.
5. 목록에 **미완료 과제/항목**이 보이면 가정에서 확인하기 좋습니다.

---

### 4.7 특정 학생을 깊게 보고 싶을 때 (선생님)

1. **반 관리 → 학생 목록** 또는 검색 흐름으로 학생을 고릅니다.
2. **학생 대시보드**(`/students/[id]`)로 들어갑니다.
3. 상단 카드에서 **완료율·출석률·최근 점수**를 봅니다.
4. 탭을 전환합니다.  
   - **점수 추이:** 기간 필터(최근 5·10·1·3개월·전체 등).  
   - **수업 이력:** 페이지네이션 목록.  
   - **알림톡:** 발송 채널·성공/실패·문자 미리보기 일부.
5. **AI 분석** 블록에서 오늘 기준 캐시된 분석을 불러오거나, **다시 분석**으로 갱신합니다(`refresh` 쿼리).

**주의:** 여기서 보는 AI는 **AI 조교(`/ai`) 설정과 별개**입니다. 학부모 문자 톤을 바꿔도 학생 분석 문구에는 자동 반영되지 않습니다.

---

### 4.8 AI 조교(학부모 피드백)를 다듬을 때

1. `/ai` 로 이동합니다.
2. **톤 프리셋**(따뜻함/분석적/간결/사용자 지정)과 **피드백 길이**를 고릅니다.
3. **포함할 정보**(점수·과제·출결·개선점·칭찬 등)를 토글합니다.
4. **말투 샘플 분석** 기능으로 예시 문자를 넣어 스타일을 시험해 볼 수 있습니다.
5. 저장 후, 이후 **알림톡 학부모 채널**에 붙는 AI 문구에 반영됩니다(워커 처리).

---

### 4.9 반·학생 데이터를 정리·마감할 때

- **반 종료:** 반을 종료 처리하면(백엔드 `POST /classes/:id/end` 등) 새 수업 생성이 막힐 수 있습니다 — 학기 말에 사용.
- **학생 삭제:** 개인정보 이슈가 있으므로 내부 규정에 맞게 진행합니다.
- **엑셀 백업:** 수업별 문자열을 `GET /lessons/:id/export` 규칙에 맞춰 내려받아 보관합니다.

---

### 4.10 로그인이 풀리거나 오류가 날 때

- **반복적인 401:** 세션 만료 가능 — 다시 로그인합니다.
- **네트워크 시간 초과:** API URL(`NEXT_PUBLIC_API_URL`)과 방화벽을 확인합니다.
- **Swagger로 확인:** 백엔드 `/api-docs`에서 동일 계정으로 엔드포인트가 살아 있는지 봅니다.

---

### 4.11 운영·개발 담당자에게 전달할 체크리스트

| 항목 | 사용자가 확인할 수 있는 방법 |
|------|------------------------------|
| 실제 알림톡 연동 여부 | 알림톡 설정 화면의 **전달 모드** 표시(mock/live) |
| 학부모 링크 유효 시간 | 서버 `PARENT_LINK_TTL_HOURS` (UI에 저장 필드 없음) |
| AI 대기 시간 | 학부모 화면에서 로딩/안내 문구, 약 3분 한도 안내 |
