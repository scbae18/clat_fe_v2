==============================================================
CLAT 챗봇 지식베이스 — 선생님 전용 지원 봇
버전: 1.0.0 | 작성일: 2026-05-05
==============================================================

이 지식베이스는 CLAT 서비스를 사용하는 선생님(학원 강사·방문 강사)이
"어떻게 해야 할지 모를 때" 챗봇에 질문하면 정확하게 답할 수 있도록
모든 기능·정책·에러·해결법을 빠짐없이 담았습니다.

챗봇 운영 원칙
- 이 문서에 없는 사실은 추측하지 말고 "운영팀에 문의"로 안내한다.
- 계정 데이터(전화번호·토큰·비밀번호)는 재전송·저장을 권하지 않는다.
- 법적 분쟁·환불·민원 대응은 전문가에게 위임하도록 안내한다.
- 카카오 비즈니스 채널 연동 절차는 카카오 정책이 수시로 바뀌므로 단정하지 않는다.
- 선생님이 어느 화면에서 막혔는지 먼저 파악한 뒤 3단계 이내로 안내한다.

==============================================================
[SECTION 1] 제품 정체성 · 용어집
==============================================================

서비스 이름: CLAT
주요 사용자:
  - 선생님(방문 강사·학원 강사): 웹 앱에 로그인해 반·학생·수업·알림톡·출결을 관리.
  - 학부모: 카카오 등으로 받은 일회성 링크(/parent/[token])로 수업 요약 확인.
  - 학생: 출결 링크(/check/[sessionId]?studentId=숫자)로 코드 입력 후 출결 체크.

핵심 기능 목록:
  반·학생 등록, 수업 템플릿 설계, 수업별 기록·저장, 알림톡 발송(모의/실연동),
  발송 내역 확인, 실시간 코드 출결, 학부모 대시보드, 선생님용 학생 AI 분석,
  학부모 피드백용 AI 조교.

용어집:
  반(Class): 학생 그룹. 학원명·반 이름·요일 스케줄로 구성.
  학생(Student): 이름, 학생 전화, 학부모 전화, 학교명. 반 다중 소속 가능.
  수업 템플릿(Lesson Template): 매 수업의 항목 구성·순서·문자 포함 여부를 미리 설계한 틀.
  템플릿 항목(Item): 공통 항목(반 전체 동일) 또는 개별 항목(학생별 다른 값).
    항목 유형: TEXT(텍스트), SELECT(선택형), COMPLETE(완료형/과제), ATTENDANCE(출결), SCORE(점수).
  수업 레코드(Lesson Record): 특정 날짜+반에 템플릿을 적용한 하나의 수업 인스턴스.
  알림톡 배치(AlimtalkBatch): 한 번의 "보내기" 작업 단위. LESSON(수업 결과) 또는 ATTENDANCE(출결 안내) 타입.
  채널 집계: 학생 번호 1통 + 학부모 번호 1통이면 total_count 2. 번호 없으면 해당 채널 미발송.
  학부모 대시보드 토큰: URL의 token값. 재발송 시 이전 토큰 즉시 무효화.
  출결 세션: 수업당 이론상 1회. 4자리 코드·만료 시각·학생별 공개 URL 생성.
  PRESENT / LATE / ABSENT: 출결 상태(출석/지각/결석).
  모의(mock) 전송: 외부 카카오 API 미호출. DB·내역·토큰은 정상 적재.
  AiSetting: 학부모 알림톡에 붙는 AI 피드백 설정. 학생 대시보드 AI와 완전히 별개.
  완료형 항목: COMPLETE 타입. 학생이 과제 완료 여부를 추적. 미완료가 대시보드에 쌓임.

==============================================================
[SECTION 2] 화면·메뉴 전체 지도
==============================================================

선생님 앱 사이드바 메뉴:

  홈 (/home)
    - 단계별 시작 가이드(일러스트·설명)만 있음. 데이터 직접 입력하는 곳 아님.

  수업 입력 (/lesson, /lesson/new, /lesson/[id])
    - 주간 캘린더, 수업 생성·상세 입력·저장·미리보기·엑셀 내보내기·알림톡 발송·출결 UI.

  학생·반 관리 (/management, /management/[반ID])
    - 반 생성·수정·종료, 학생 목록·추가·엑셀 일괄 업로드·반에서 학생 제외.

  학생 상세 (/students/[id])
    - 사이드바에 직접 메뉴는 없음. 반 관리 화면에서 학생 클릭 시 진입.
    - 완료율·출석률·최근 점수, 점수 추이, 수업 이력, 알림톡 이력, 선생님용 AI 분석.

  수업 템플릿 (/template, /template/new, /template/[id], /template/[id]/edit)
    - 항목 설계·순서 변경(드래그앤드롭)·미리보기.

  알림톡 (/alimtalk)
    - 인트로·아웃트로 문구 편집·저장. 전달 모드(mock/live) 표시.

  알림톡 발송 내역 (/alimtalk/history)
    - 배치 목록·필터·상세. 건별 성공/실패·학부모 토큰·AI 피드백 포함 여부.

  AI 조교 (/ai)
    - 학부모 피드백 AI 설정(말투·길이·포함 정보)·말투 샘플 분석.

  내 정보 (/me)
    - 프로필·비밀번호 변경·회원 탈퇴.

  로그인·가입 (/login, /signup)

공개 화면 (로그인 없이 접근 가능):
  /check              → 개발·테스트용 출결 진입(세션 ID·학생 ID 직접 입력)
  /check/[sessionId]?studentId=숫자 → 학생 출결 화면
  /parent/[token]     → 학부모 대시보드

주의: /find-password(비밀번호 찾기) 화면은 현재 코드에 구현이 확인되지 않음.
  → "비밀번호를 잊었다"는 질문이 오면: 운영팀에 계정 초기화 요청 안내.

==============================================================
[SECTION 3] 기능별 상세 동작
==============================================================

--------------------------------------------------------------
3-1. 계정·인증
--------------------------------------------------------------

회원가입:
  - 이메일+비밀번호 입력 후 가입 완료.
  - 이메일 중복 시 오류 발생.

로그인:
  - 이메일+비밀번호로 로그인.
  - 토큰은 브라우저 쿠키에 저장됨. 만료 시 자동 갱신 시도, 실패하면 로그인 화면으로 이동.

로그아웃:
  - 서버에서 토큰 무효화 후 FE 토큰 삭제.

내 정보 수정:
  - 이름·이메일 변경 가능. 이메일 변경 시 현재 비밀번호 입력 필요.
  - 비밀번호 변경 가능.

회원 탈퇴:
  - 즉시 삭제 아님. 30일 유예 후 삭제 정책.

--------------------------------------------------------------
3-2. 반(Class) 관리
--------------------------------------------------------------

반 생성:
  - 학원명, 반 이름, 요일(다중 선택) 입력.
  - 이름만으로도 생성 가능. 이후 학생 추가.

반 수정·삭제:
  - 수정: 반 이름·요일·학원명 변경.
  - 삭제: 데이터 연계에 주의. 운영 중 반은 종료 처리 권장.

반 종료 (POST /classes/:id/end):
  - 종료된 반은 새 수업 생성이 제한될 수 있음.
  - 학기/분기 마감 시점에만 사용.

반 목록 필터:
  - status=active: 진행 중 반
  - status=ended: 종료 반

반에 학생 추가:
  - 반 상세에서 개별 추가 또는 학생 엑셀 업로드 후 일괄 연결.
  - 학생을 반에서 제외해도 학생 자체는 삭제되지 않음.

--------------------------------------------------------------
3-3. 학생 등록·관리
--------------------------------------------------------------

개별 등록:
  - 이름·학생 전화·학부모 전화·학교명 입력.
  - 학부모 전화는 학부모 알림톡 발송에 필수. 없으면 학부모 채널 발송 불가.
  - 생성 시 class_ids로 반 즉시 연결 가능.

엑셀 일괄 업로드 (POST /students/bulk):
  - multipart/form-data로 파일 전송.
  - 응답: 성공 건수, 실패 건수, 실패 행·이유 목록(errors[]).
  - 실패 행만 수정해서 재업로드 가능.

학생 수정:
  - 이름·전화·학교명 등 수정 가능.
  - 저장 후 새로고침으로 반영 확인 권장.

학생 삭제:
  - 삭제 전 엑셀 export로 데이터 백업 강력 권장.
  - 수업 기록과의 연계 처리는 서버 정책에 따름.

학생 상세 통계 (학생 대시보드 /students/[id]):
  - 이번 달 완료율·출석률·최근 점수 카드.
  - 점수 추이 탭: 기간 필터 recent5/recent10/1month/3month/all.
  - 수업 이력 탭: 페이지네이션 목록.
  - 알림톡 이력 탭: 배치 타입·mock/live 여부·성공/실패·미리보기.
  - AI 분석 섹션: 오늘 기준 캐시된 분석. "다시 분석" 버튼으로 갱신 가능.
  - 미완료 항목: COMPLETE 타입 미완료 목록. 화면에서 바로 완료 처리 가능.

--------------------------------------------------------------
3-4. 수업 템플릿
--------------------------------------------------------------

역할:
  매 수업마다 반복 입력할 항목(필드)을 미리 설계하는 틀.

항목 구분:
  공통 항목: 반 전체가 동일한 값 (예: 오늘 수업 주제, 숙제 내용).
  개별 항목: 학생마다 다른 값 (예: 점수, 출결, 개인 피드백).

항목 유형 설명:
  TEXT      : 자유 텍스트 입력.
  SELECT    : 선택지 중 하나 고르기.
  COMPLETE  : 완료/미완료 체크. 과제 추적에 활용.
              미완료 항목은 학생 대시보드와 학부모 페이지에 표시됨.
  ATTENDANCE: 출결 전용. 템플릿당 반드시 정확히 1개만 허용.
  SCORE     : 점수 입력. "85" 또는 "85/100" 형식 모두 허용.
              반 평균·최고점은 서버가 자동 계산.

핵심 정책 — ATTENDANCE 항목 수:
  템플릿당 ATTENDANCE 항목이 0개 또는 2개 이상이면
  출결 세션 시작 시 422 오류(NO_ATTENDANCE_ITEM) 발생.
  → 반드시 정확히 1개만 설정해야 함.

문자 포함 설정:
  각 항목별로 "알림톡 문자에 포함할지 여부" 지정 가능.

템플릿 미리보기:
  편집 화면에서 학부모용 문자 형태를 대략 확인 가능.

템플릿 수정 주의:
  이미 생성된 수업 레코드는 수정된 템플릿이 자동 반영되지 않을 수 있음.
  기존 수업 화면에서 TEMPLATE_NOT_FOUND 오류가 나면 수업 재생성 필요.

추천 최소 구성:
  공통 텍스트 1개 + 출결(ATTENDANCE) 1개 + 점수(SCORE) 1개 + 과제(COMPLETE) 1개.

--------------------------------------------------------------
3-5. 수업 입력 (매일 기록)
--------------------------------------------------------------

수업 생성 흐름:
  1) /lesson 캘린더에서 날짜 선택.
  2) 반·템플릿·날짜 선택 후 수업 초안(DRAFT) 생성.
  3) 공통 블록(오늘 진도 등) 입력.
  4) 학생별 표에서 출결·점수·과제 완료·선택 등 입력.
  5) 자주 저장 권장 (저장 안 하면 데이터 유실 가능).

저장:
  수업 중 [저장] 버튼으로 수시 저장. 저장 후 완료율 표시.

미리보기:
  수업 데이터 기반 학부모/학생 수신 문자 내용 확인 가능.

엑셀 내보내기:
  수업 데이터를 엑셀로 export.
  구조: 상단 학부모 전화번호 블록 / 빈 행 / 하단 학생 전화번호 블록.
  헤더: 전화번호 | 문자내용 | 학생명 (3열 고정).
  학부모 번호 없는 학생은 상단 블록에서 생략됨.

점수 입력 형식:
  "85" 또는 "85/100" 둘 다 허용.
  반 평균·최고점은 서버가 자동 계산.

완료형 항목:
  학생별 COMPLETE 항목 체크. 미완료 항목이 학생 대시보드에 누적·표시됨.

--------------------------------------------------------------
3-6. 알림톡 발송
--------------------------------------------------------------

발송 방법:
  1) 수업 입력 화면에서 발송할 학생 선택.
  2) [알림톡 발송] 버튼 클릭.
  3) 결과 팝업에서 성공/실패 건수와 mock 여부 확인.
  4) /alimtalk/history에서 배치 단위 상세 재확인 가능.

채널 집계 방식:
  학생 번호로 1건 + 학부모 번호로 1건 = 총 2건 집계.
  번호가 없으면 해당 채널은 발송되지 않고 집계에서 제외.

재발송:
  같은 수업·같은 학생에게 다시 발송하면 이전 학부모 링크 즉시 무효화.
  학부모에게는 가장 최근에 받은 링크만 열도록 안내 필요.

발송 대상 없이 보내면:
  STUDENT_IDS_REQUIRED 오류(400). 최소 1명 이상 선택 필요.

반에 없는 학생 포함하면:
  STUDENT_NOT_IN_LESSON 오류(422).

알림톡 설정 (인트로·아웃트로):
  /alimtalk 메뉴에서 편집.
  사용 가능한 변수: {강사명}, {학원명}, {학생이름}, {날짜}, {반이름}.

문자 구조:
  인트로 + 본문(수업 데이터) + 아웃트로 + 학부모에게만 대시보드 링크.
  SCORE 항목은 "N점 (최고점: …, 반 평균: …)" 형식으로 표시.

전달 모드 (delivery_mode):
  mock: 외부 카카오 미호출. 내부 DB·배치·토큰만 기록. 발송 내역은 정상 생성.
  live: 실제 카카오 발송. 서버 환경변수 ALIMTALK_DELIVERY_MODE 변경으로 전환.
  → mock에서 도착이 없다고 오류가 아님. 정상 동작.

발송 내역 상세:
  배치 목록에서 타입(LESSON/ATTENDANCE), 건별 성공/실패, 학부모 토큰, AI 피드백 포함 여부 확인 가능.

--------------------------------------------------------------
3-7. 실시간 출결 (코드 입력 방식)
--------------------------------------------------------------

[선생님 측 흐름]

출결 시작:
  1) 해당 수업 상세 화면의 출결 플로팅 UI 사용.
  2) 제한 시간 선택 (프리셋: 5·10·15·20분 또는 직접 입력).
  3) [출결 시작] 클릭 → 4자리 코드 + 학생별 접속 URL 생성.
  4) 알림톡 또는 단체 채팅방으로 링크·코드 공유.

모니터링:
  화면에서 출석·지각·결석 인원과 학생별 상태 실시간 확인.

수기 수정:
  특정 학생의 상태를 선생님이 직접 변경 가능.
  API: PATCH /attendance/sessions/:sessionId/students/:studentId
  body: { status: PRESENT | LATE | ABSENT }

출결 종료:
  [종료] 버튼 클릭 → 수업 레코드의 출결 칸에 결과 자동 반영.
  수동 종료가 가장 안전. 자동 만료는 서버 처리에 의존.

출결 세션 정책:
  수업당 1회만 가능. 한 번 종료하거나 만료된 세션은 다시 시작 불가.
  재시작 시도 시 SESSION_ALREADY_EXISTS(409) 오류.
  → 이미 종료된 경우: 수기 출결로 보정.

알림 발송 실패해도 세션은 유지:
  알림톡이 실패해서 배치가 없어도 출결 세션 자체는 살아 있음.
  → 링크·코드를 단체 채팅방 등 다른 채널로 직접 공유해서 출결 진행 가능.

출결 링크 형식:
  /check/{sessionId}?studentId={학생ID}
  앱 배포 도메인(NEXT_PUBLIC_APP_ORIGIN)이 설정되면 full URL 자동 생성.

[학생 측 흐름]

접속:
  선생님이 공유한 개인 링크(/check/[sessionId]?studentId=숫자) 또는
  테스트 진입(/check)에서 세션 ID·학생 ID 직접 입력.

코드 입력:
  4자리 코드 입력 → 결과 확인.

지각 판정 기준:
  세션 시작 후 10분 초과 입력 → 지각(LATE) 처리.
  단, 전체 제한 시간이 5분 이하인 경우 지각 없이 출석(PRESENT)만.

중복 제출:
  이미 체크한 학생이 다시 입력 시 ALREADY_CHECKED_IN(409). 정상 동작.

출결 화면 오류 대응표:
  "코드가 틀렸다"     → 선생님 화면의 4자리 코드와 대조.
  "시간이 지났다"     → expires_at 경과. 선생님에게 수기 반영 요청.
  "이미 처리됐다"     → 정상. 결과 화면 확인.
  "세션이 종료됐다"   → 선생님이 수기 상태 수정.
  "세션을 찾을 수 없다" → sessionId 또는 studentId 확인.

--------------------------------------------------------------
3-8. 학부모 공개 대시보드 (/parent/[token])
--------------------------------------------------------------

역할:
  알림톡(학부모 채널)에 실린 일회성 링크를 통해
  해당 수업의 요약·점수·항목·미완료 목록·최근 수업 목록 확인.

접근 방법:
  카카오 등으로 받은 URL 탭 → 별도 로그인 불필요.

표시 내용:
  학생·반·학원명, 수업일, 출결 상태, 점수, 추가 항목, AI 코멘트,
  미완료 과제 목록, 최근 수업 목록.

AI 코멘트:
  비동기 생성. 생성 중이면 "피드백을 준비 중이에요" 문구 표시.
  완료되면 본문, 실패하면 실패 안내 표시.
  → 비어 있으면: 잠시 후 다시 열기 안내. 최대 약 3분 대기.

링크 유효 시간:
  서버 환경변수 PARENT_LINK_TTL_HOURS 기준 (UI에서 변경 불가).

링크 오류:
  "잘못된 토큰" → 404 INVALID_TOKEN. 링크 오전달 또는 URL 손상.
  "만료됐다"    → 410 TOKEN_EXPIRED. TTL 경과 또는 재발송으로 무효화.
  → 두 경우 모두: 선생님에게 재발송 요청.

재발송 시 주의:
  재발송하면 새 토큰이 발급되고 이전 토큰은 즉시 무효.
  학부모는 항상 최신 문자의 링크를 사용해야 함.

--------------------------------------------------------------
3-9. AI 기능 — 두 가지는 완전히 다름 (혼동 금지)
--------------------------------------------------------------

A) AI 조교 설정 (/ai 메뉴, API /ai-settings)
  목적: 학부모 알림톡에 붙는 AI 피드백 스타일 설정.
  말투 프리셋: WARM(따뜻함), ANALYTICAL(분석적), CONCISE(간결), CUSTOM(사용자 지정).
  피드백 길이: SHORT, MEDIUM, LONG.
  포함 토글: 점수, 과제(숙제), 출결, 개선점, 칭찬 등.
  데이터 기간: THIS_LESSON, RECENT_3, RECENT_5, RECENT_1MONTH.
  말투 샘플 분석: 예시 문장을 입력하면 AI가 스타일 분석. 원장·학원 말투를 맞출 때 활용.
  저장 후 이후 발송되는 알림톡 학부모 피드백에 반영.

B) 학생 대시보드 AI 분석 (/students/[id] 화면)
  목적: 선생님이 해당 학생의 종합 분석을 보는 기능.
  AI 조교 설정(/ai)과 완전히 별개 시스템. 서버 별도 프롬프트·일 캐시 사용.
  "다시 분석" 버튼으로 오늘 캐시 무시하고 재생성 가능.
  캐시 기준: 사용자+학생+날짜 단위, TTL 86400초(하루).

핵심 구분 정리:
  /ai 설정 변경 → 학부모 알림톡 피드백 문체에만 영향.
  /ai 설정 변경 → 학생 대시보드 AI 분석에 자동 반영되지 않음. (설계상 정상)

==============================================================
[SECTION 4] 온보딩 — 처음 시작하는 선생님 안내
==============================================================

단계별 순서:

1단계: 회원가입 후 로그인.

2단계: 반 만들기.
  /management → [반 만들기] → 반 이름 입력 (이름만으로 시작 가능).

3단계: 학생 등록.
  반 상세 화면에서 개별 추가 또는 엑셀 일괄 업로드.
  학부모 알림톡을 쓸 계획이면 학부모 전화번호 필수 입력.

4단계: 수업 템플릿 만들기.
  /template → [새 템플릿] → 항목 추가.
  출결 앱 사용 예정: ATTENDANCE 항목 정확히 1개 추가.
  과제 추적 예정: COMPLETE 항목 추가.
  알림톡에 넣을 항목: include_in_message 체크.

5단계: 알림톡 인트로·아웃트로 설정.
  /alimtalk → 인트로·아웃트로 입력 → 저장.

6단계: AI 조교 설정 (선택).
  /ai → 말투·길이·포함 정보 설정 → 저장.

7단계: 첫 수업 입력.
  /lesson → 날짜 선택 → 반·템플릿 선택 → 수업 생성 → 내용 입력 → 저장.

8단계: 알림톡 발송.
  수업 화면에서 학생 선택 → [알림톡 발송] → 결과 확인.

자주 막히는 상황:
  "출결 시작이 안 돼요" → 템플릿에 ATTENDANCE 항목이 0개이거나 2개 이상. 템플릿 수정 필요.
  "발송 오류가 나요"   → 발송 대상을 아무도 선택 안 했을 수 있음. 최소 1명 이상 선택.
  "학부모 링크가 안 열려요" → 재발송으로 이전 링크 무효화됐을 수 있음. 최신 링크 안내.

==============================================================

에러 코드 | HTTP | 의미 | 대응
NO_ATTENDANCE_ITEM      | 422 | 출결 항목 없음 또는 2개 이상 | 템플릿 편집에서 ATTENDANCE 항목 정확히 1개로 수정
SESSION_ALREADY_EXISTS  | 409 | 출결 세션 재시작 불가 | 수기 출결로 보정
INVALID_ATTENDANCE_CODE | 400 | 코드 불일치 | 선생님 화면 4자리와 대조
SESSION_EXPIRED         | 422 | 세션 만료 | 선생님이 수기 PATCH 처리
SESSION_ENDED           | 422 | 선생님이 세션 종료 | 수기 출결 수정
ALREADY_CHECKED_IN      | 409 | 이미 출결 완료 | 정상 — 결과 화면 확인
STUDENT_IDS_REQUIRED    | 400 | 발송 대상 없음 | 학생 최소 1명 선택
STUDENT_NOT_IN_LESSON   | 422 | 수업에 없는 학생 | 해당 반 학생만 선택
INVALID_TOKEN           | 404 | 학부모 토큰 무효 | 링크 재발송
TOKEN_EXPIRED           | 410 | 학부모 링크 만료 | 링크 재발송
TEMPLATE_NOT_FOUND      | —   | 수업이 참조하는 템플릿 문제 | 템플릿 재연결 또는 수업 재생성
NOT_FOUND               | 404 | 리소스 없음 | 경로·ID 재확인
FORBIDDEN               | 403 | 소유권 없음 | 본인 계정 데이터인지 확인
(인증 오류)             | 401 | 토큰 만료·인증 실패 | 재로그인

공통 오류 응답 형식:
  { success: false, error: { code: "오류코드", message: "설명" } }

==============================================================
[SECTION 6] 트러블슈팅 매트릭스 (전체)
==============================================================

[로그인·인증]
증상: 로그인 직후 튕김
원인: accessToken 쿠키 차단, 쿠키 미설정
해결: 브라우저 쿠키 허용 설정 → 재로그인. 시크릿 모드·다른 브라우저로도 테스트.

증상: 401 오류가 반복됨
원인: refresh 토큰 만료, 서버 시간 불일치, 토큰 탈취 의심
해결: 재로그인. 반복되면 비밀번호 변경 후 재로그인.

증상: 비밀번호 찾기 메뉴가 없음
원인: 현재 미구현
해결: 운영팀에 계정 초기화 요청 안내.

증상: 이메일 변경 시 비밀번호 요구
원인: 보안 정책 (PATCH /auth/me 설계)
해결: 현재 비밀번호 입력 → 정상 절차.

[템플릿·출결 시작]
증상: 출결 시작 버튼이 눌리지 않거나 422 오류
원인: 템플릿에 ATTENDANCE 항목이 0개 또는 2개 이상
해결: /template → 해당 템플릿 편집 → ATTENDANCE 항목 정확히 1개로 수정.

증상: 출결 세션 재시작 시 409 오류
원인: 이미 활성/종료 세션 존재 (SESSION_ALREADY_EXISTS)
해결: 수기 출결로 상태 보정. 새 수업 레코드 생성은 운영 규칙 확인 후 결정.

증상: 출결 알림 발송 실패 → 배치 내역에 출결 배치 없음
원인: 알림 발송 실패 시 출결용 배치 미생성 정책 (v2)
해결: 세션은 살아 있음. 링크·코드를 단체 채팅방 등으로 직접 공유해서 진행.

[학생 출결 코드 입력]
증상: "코드가 틀리다"
원인: 코드 불일치
해결: 선생님 화면에 표시된 4자리와 정확히 대조.

증상: "시간이 지났다" 또는 SESSION_EXPIRED
원인: 만료 시각(expires_at) 경과
해결: 선생님이 수기로 해당 학생 상태 변경.

증상: "이미 처리됐다" 또는 ALREADY_CHECKED_IN
원인: 중복 제출
해결: 정상. 이미 출결 완료된 상태.

증상: "세션이 종료됐다" 또는 SESSION_ENDED
원인: 선생님이 종료 버튼 클릭
해결: 선생님이 수기로 상태 수정.

[알림톡 발송]
증상: 발송 건수와 학생 수가 안 맞음
원인: 채널 단위 집계 (학생 번호 + 학부모 번호 = 최대 2건/학생)
해결: 정상. 학부모 번호 없는 학생은 학부모 채널 발송 제외됨.

증상: mock인데 카카오 미도착
원인: delivery_mode=mock → 외부 발송 안 함
해결: 정상 동작. 실연동 필요하면 인프라 담당자에게 live 전환 요청.

증상: 발송 400 오류 (STUDENT_IDS_REQUIRED)
원인: 발송 대상 학생 미선택
해결: 최소 1명 이상 선택 후 재시도.

증상: 발송 422 오류 (STUDENT_NOT_IN_LESSON)
원인: 수업에 포함되지 않은 학생 선택
해결: 해당 반 학생만 선택.

[학부모 링크]
증상: 학부모가 "링크가 만료됐대요"
원인: TTL 경과 또는 재발송으로 이전 토큰 무효화
해결: 수업 화면에서 해당 학생에게 재발송. 학부모에게 새 문자 링크 사용 안내.

증상: 학부모가 다른 학생 정보가 보인다고 함
원인: 잘못된 링크가 전달됨
해결: 즉시 재발송하여 올바른 링크 전달. 잘못 전달된 링크는 재발송으로 자동 무효화.

증상: 학부모 페이지 AI 코멘트가 비어 있음
원인: AI 생성 중(pending 상태)
해결: 잠시 후(약 3분 이내) 다시 열기 안내.

[수업 입력·저장]
증상: 저장 안 하고 화면 이탈 → 데이터 유실
원인: 자동 저장 없음
해결: 입력 중 자주 [저장] 클릭 습관화.

증상: TEMPLATE_NOT_FOUND 오류
원인: 수업이 참조하는 템플릿이 삭제됨
해결: 템플릿 재연결 또는 수업 재생성.

[엑셀]
증상: 엑셀에 같은 학생이 두 번 나옴
원인: 학부모 블록(상단) + 학생 블록(하단) 구조
해결: 정상. 상단은 학부모 전화, 하단은 학생 전화.

증상: 특정 학생이 상단에 없음
원인: 학부모 전화번호 미등록
해결: 학생 정보에서 학부모 전화번호 추가 후 재export.

[AI 조교]
증상: AI 조교 설정 바꿨는데 학생 대시보드 AI 분석이 안 바뀜
원인: 두 AI는 완전히 별개 시스템
해결: 정상 동작. 학생 분석은 /ai-settings와 무관.

[API·시스템]
증상: Swagger는 열리는데 화면이 실패
원인: NEXT_PUBLIC_API_URL 환경변수 오설정 또는 CORS·쿠키 정책 문제
해결: 인프라 담당자에게 환경변수·CORS 설정 확인 요청.

증상: 오늘만 전체 API가 느림
원인: 서버 부하·네트워크 이슈 가능성
해결: 우선 저장 최소화. 상태 페이지·인프라 로그 확인.

증상: 개발 환경과 운영 환경 데이터가 다름
원인: DB 환경 분리 (정상)
해결: 환경 혼동 여부 먼저 확인.

==============================================================
[SECTION 7] 선생님 Q&A 전체 (대량 FAQ)
==============================================================

--- [계정·시작] ---

Q: 처음 쓰는데 어디부터 해야 하나요?
A: 순서는 반 생성 → 학생 등록 → 템플릿 생성 → 수업 생성·입력 → 알림톡 발송입니다.
   홈(/home) 화면은 참고용 가이드이고 실제 작업은 각 메뉴에서 진행합니다.

Q: 로그인했는데 다시 로그인 화면으로 튕겨요.
A: 브라우저에서 쿠키가 차단됐을 가능성이 큽니다.
   쿠키 허용 설정 후 재로그인하세요. 그래도 반복되면 시크릿 모드나 다른 브라우저로 확인해 보세요.

Q: 비밀번호를 잊었어요. 찾기 기능이 어디 있나요?
A: 현재 비밀번호 찾기 화면은 제공되지 않습니다. 운영팀에 계정 초기화를 요청해 주세요.

Q: 이메일을 바꾸려는데 왜 비밀번호를 또 입력해야 하나요?
A: 이메일 변경 시 본인 확인을 위해 현재 비밀번호 입력이 필요합니다. 보안 정책입니다.

Q: 회원 탈퇴하면 바로 삭제되나요?
A: 아닙니다. 탈퇴 신청 후 30일 유예 기간이 있습니다. 그 이후 영구 삭제됩니다.

Q: 탈퇴 신청 후 마음이 바뀌었어요. 취소할 수 있나요?
A: 유예 기간 내 취소 가능 여부는 운영팀에 확인하세요.

Q: 공용 PC에서 사용했어요. 어떻게 정리하나요?
A: 반드시 로그아웃 후 브라우저 쿠키와 캐시를 삭제하세요.

--- [반 관리] ---

Q: 반을 먼저 만들어야 하나요, 학생을 먼저 만들어야 하나요?
A: 반을 먼저 만드는 게 편합니다. 학생 생성 시 바로 반에 연결할 수도 있습니다.

Q: 반을 종료하면 무슨 일이 생기나요?
A: 종료된 반은 새 수업 생성이 제한될 수 있습니다. 학기·분기 마감 시점에만 사용하세요.

Q: 종료한 반을 다시 활성화할 수 있나요?
A: 운영팀에 확인하세요. 일반적으로 새 반 생성을 권장합니다.

Q: 반에서 학생을 제외하면 학생 자체도 삭제되나요?
A: 아닙니다. 반 소속에서만 제외되고 학생 데이터는 유지됩니다.

Q: 반 목록에서 active/ended 필터는 뭔가요?
A: active는 진행 중 반, ended는 종료 반입니다.

Q: 반 이름에 학원명을 꼭 입력해야 하나요?
A: 필수는 아닙니다. 이름만으로 반 생성이 가능합니다.

--- [학생 등록] ---

Q: 학생 전화번호와 학부모 전화번호 둘 다 필요한가요?
A: 필수 여부는 화면 검증에 따르지만, 학부모 알림톡을 보내려면 학부모 전화번호가 있어야 합니다.

Q: 엑셀 업로드 실패 행은 어디서 확인하나요?
A: 업로드 결과 응답의 errors[] 항목에 실패 행 번호와 이유가 표시됩니다.
   해당 행만 수정해서 재업로드하면 됩니다.

Q: 학생 삭제하면 수업 기록도 같이 지워지나요?
A: 데이터 보존 정책은 서버 구현에 따릅니다. 삭제 전 엑셀 export로 백업을 강력 권장합니다.

Q: 학생이 여러 반에 속할 수 있나요?
A: 가능합니다. 학생은 다중 반 소속이 지원됩니다.

Q: 학생 정보 수정 후 반영이 느려요.
A: 저장 직후 캐시가 남아 있을 수 있습니다. 새로고침 후 확인하세요.

--- [템플릿] ---

Q: 템플릿에 어떤 항목을 넣어야 하나요?
A: 최소 권장 구성: 출결(ATTENDANCE) 1개 + 점수(SCORE) 1개 + 과제(COMPLETE) 1개 + 공통 텍스트 1개.

Q: 출결 항목을 2개 넣으면 왜 안 되나요?
A: ATTENDANCE 항목은 템플릿당 정확히 1개만 허용됩니다.
   0개이거나 2개 이상이면 출결 세션 시작 시 422 오류가 납니다.

Q: 점수 입력은 어떻게 하나요?
A: "85"만 입력해도 되고, "85/100"처럼 획득점수/만점 형식도 가능합니다.
   반 평균·최고점은 서버가 자동 계산합니다.

Q: 문자에 넣을 항목을 어떻게 선택하나요?
A: 템플릿 항목 편집 시 "알림 문자에 포함" 설정을 켜면 됩니다.

Q: 완료형(COMPLETE) 항목은 어디에 쓰나요?
A: 과제 완료 여부 추적에 씁니다. 미완료 항목이 학생 대시보드와 학부모 페이지에 표시됩니다.

Q: 템플릿 수정 후 기존 수업에도 반영되나요?
A: 이미 생성된 수업 레코드에는 자동 반영되지 않을 수 있습니다. 기존 수업은 별도 확인 필요.

Q: 템플릿 항목 순서를 바꿀 수 있나요?
A: 네. 드래그앤드롭으로 순서 변경이 가능합니다.

--- [수업 입력] ---

Q: 수업은 자동으로 생성되나요?
A: 날짜·반·템플릿을 선택해서 직접 생성해야 합니다. /lesson 캘린더에서 날짜 선택 후 생성하세요.

Q: 저장 안 하면 입력한 내용이 날아가나요?
A: 네. 자동 저장이 없으므로 수시로 [저장] 버튼을 눌러야 합니다.

Q: 완료율은 어떻게 계산되나요?
A: 공통·개별 항목 입력 상태와 완료형 항목 체크 상태를 기반으로 계산됩니다.

Q: TEMPLATE_NOT_FOUND 오류가 떠요.
A: 수업이 참조하는 템플릿이 삭제됐거나 연결이 끊어진 경우입니다.
   템플릿을 다시 연결하거나 수업을 재생성하세요.

Q: 수업 날짜를 변경할 수 있나요?
A: 수업 수정 API로 변경 가능합니다. 데이터 일관성을 위해 운영 규칙을 함께 확인하세요.

Q: 공통 항목과 개별 항목의 차이가 뭔가요?
A: 공통 항목은 반 전체가 동일한 값(예: 오늘 진도), 개별 항목은 학생마다 다른 값(예: 점수)입니다.

--- [알림톡 발송] ---

Q: 발송 건수가 왜 학생 수보다 많아요?
A: 학생 번호 1건 + 학부모 번호 1건 = 최대 2건으로 집계되기 때문입니다.

Q: 특정 학생만 선택해서 보낼 수 있나요?
A: 네. 발송 화면에서 원하는 학생만 선택 후 발송 가능합니다.

Q: 아무도 선택 안 하고 발송하면 어떻게 되나요?
A: 400 오류(STUDENT_IDS_REQUIRED)가 발생합니다. 최소 1명 이상 선택해야 합니다.

Q: 같은 학생에게 같은 수업으로 다시 보내면요?
A: 이전에 발송된 학부모 링크는 즉시 무효화되고 새 링크가 발급됩니다.
   학부모에게 최신 링크를 사용하도록 안내하세요.

Q: mock 상태인데 도착이 안 해요. 오류인가요?
A: 정상입니다. mock 모드에서는 카카오로 실제 발송하지 않습니다.
   내역·DB는 정상 기록됩니다. 실연동은 인프라에서 live로 전환해야 합니다.

Q: 발송 내역은 어디서 보나요?
A: /alimtalk/history 메뉴에서 배치 단위로 확인 가능합니다.

Q: 인트로·아웃트로에 학원 이름을 자동으로 넣고 싶어요.
A: {학원명} 변수를 사용하면 자동 치환됩니다.
   사용 가능 변수: {강사명}, {학원명}, {학생이름}, {날짜}, {반이름}.

--- [출결] ---

Q: 출결 시작이 안 돼요.
A: 가장 흔한 원인은 템플릿에 ATTENDANCE 항목이 없거나 2개 이상인 경우입니다.
   /template에서 해당 템플릿을 편집해 정확히 1개로 맞추세요.

Q: 출결 제한 시간은 몇 분으로 하는 게 좋나요?
A: 10~15분을 일반적으로 권장합니다.
   5분 이하로 설정하면 지각 판정 없이 출석만 처리됩니다.

Q: 학생이 10분 넘게 지나서 코드를 넣으면 어떻게 되나요?
A: 자동으로 지각(LATE) 처리됩니다.
   단, 제한 시간이 5분 이하인 경우 지각 없이 출석(PRESENT)으로만 처리됩니다.

Q: 같은 수업에서 출결을 두 번 할 수 있나요?
A: 설계상 수업당 1회입니다. 재시작 시도 시 SESSION_ALREADY_EXISTS(409) 오류가 납니다.
   필요하면 해당 학생을 수기로 상태 변경하세요.

Q: 선생님이 직접 출석·결석을 바꿀 수 있나요?
A: 가능합니다. 수업 화면에서 학생 행의 상태를 수기로 변경할 수 있습니다.

Q: 출결 알림이 실패했는데 출결은 진행할 수 있나요?
A: 네. 알림 실패로 배치가 없어도 세션은 유지됩니다.
   링크·코드를 단체 채팅방 등 다른 채널로 직접 공유해서 진행하세요.

Q: 출결 종료를 안 하면 어떻게 되나요?
A: 만료 시각이 지나면 서버가 자동 처리할 수 있으나, 수동 종료가 가장 안전합니다.

Q: 학생에게 어떤 링크를 보내야 하나요?
A: /check/{sessionId}?studentId={학생ID} 형식의 개인 링크입니다.
   출결 세션 시작 후 학생별 링크가 자동 생성됩니다.

Q: 학생이 코드 화면을 못 찾으면 어떻게 하나요?
A: /check 화면(테스트 진입)에서 세션 ID와 학생 ID를 직접 입력해 링크 형식을 재확인하세요.

Q: 학생이 이미 출결했는데 다시 입력하려 해요.
A: 중복 제출은 ALREADY_CHECKED_IN(409)으로 자동 차단됩니다. 이미 완료된 상태입니다.

--- [학부모 대시보드] ---

Q: 학부모가 링크를 열었는데 만료됐다고 해요.
A: TTL 만료이거나 재발송으로 이전 링크가 무효화된 경우입니다.
   수업 화면에서 해당 학생에게 다시 발송하고 새 링크를 안내하세요.

Q: 학부모 페이지에 AI 피드백이 안 보여요.
A: AI가 생성 중일 수 있습니다. 잠시 후(최대 약 3분) 다시 열어 보도록 안내하세요.

Q: 학부모가 다른 학생 정보가 보인다고 해요.
A: 링크가 잘못 전달됐을 가능성이 큽니다.
   즉시 재발송으로 올바른 링크를 전달하세요. 잘못된 링크는 재발송으로 자동 무효화됩니다.

Q: 학부모 링크의 유효 기간은 얼마나 되나요?
A: 서버 설정(PARENT_LINK_TTL_HOURS)에 따라 다릅니다. 정확한 기간은 운영팀에 확인하세요.

Q: 학부모 링크는 로그인 없이 열 수 있나요?
A: 네. 학부모 대시보드는 로그인 없이 링크만으로 접근 가능합니다.

--- [학생 대시보드 (선생님)] ---

Q: 학생 상세 화면은 어떻게 들어가나요?
A: /management(반 관리)에서 해당 학생을 클릭하면 /students/[id] 페이지로 이동합니다.

Q: 학생 AI 분석이 최신이 아닌 것 같아요.
A: 하루 단위로 캐시됩니다. "다시 분석" 버튼을 클릭하면 즉시 새로 생성됩니다.

Q: AI 조교 설정을 바꿨는데 학생 분석이 안 바뀌어요.
A: 정상입니다. 학생 대시보드 AI 분석은 AI 조교(/ai) 설정과 완전히 별개 시스템입니다.

Q: 점수 추이 그래프 기간은 어떻게 바꾸나요?
A: 탭 상단 필터에서 recent5, recent10, 1month, 3month, all 중 선택하면 됩니다.

Q: 미완료 항목 목록은 어디서 보나요?
A: 학생 대시보드 하단에 미완료 COMPLETE 항목 목록이 표시됩니다. 화면에서 바로 완료 처리도 가능합니다.

--- [AI 조교] ---

Q: 어떤 톤(말투)으로 시작하면 좋을까요?
A: WARM(따뜻함)으로 시작해 반응을 보고 ANALYTICAL(분석적)이나 CONCISE(간결)로 조정하는 걸 권장합니다.

Q: 말투 샘플 분석은 어디에 쓰나요?
A: 원장님 또는 학원 고유 말투를 AI 피드백에 적용하고 싶을 때 예시 문장을 입력해 스타일을 맞춥니다.

Q: 피드백 길이를 LONG으로 하면 무조건 길어지나요?
A: 길이 성향을 조정하는 옵션입니다. 실제 길이는 데이터 양과 프롬프트에도 영향을 받습니다.

Q: AI 조교 설정 저장 후 언제부터 반영되나요?
A: 저장 직후 다음 알림톡 발송부터 적용됩니다.

--- [엑셀] ---

Q: 엑셀에 학생이 두 번씩 나와요.
A: 정상입니다. 상단은 학부모 전화번호 블록, 하단은 학생 전화번호 블록으로 나뉜 구조입니다.

Q: 상단 블록에 없는 학생이 있어요.

   학생 정보에서 학부모 전화번호를 추가하면 다음 export에 포함됩니다.

Q: 엑셀 헤더 구조가 어떻게 되나요?
A: 전화번호 | 문자내용 | 학생명 (3열 고정).

A: DB 환경이 분리되어 있어 정상입니다. 환경 혼동 여부를 먼저 확인하세요.

[SECTION 8] 수업 당일 업무 흐름 (전체 시나리오)
==============================================================

[시나리오 1: 일반 수업 기록 + 알림톡 발송]
1. /lesson → 캘린더에서 오늘 날짜 선택.
2. 해당 반 수업 카드 → [입력하기].
3. 공통 블록(진도·메모 등) 입력.
4. 학생별 표에서 출결·점수·과제 완료 입력.
5. [저장] 클릭.
6. 발송할 학생 선택 → [알림톡 발송].
7. 결과 팝업에서 성공/실패 건수 확인.
8. 필요 시 /alimtalk/history에서 상세 확인.

[시나리오 2: 실시간 출결 포함]
1~4. 위와 동일.
5. 출결 플로팅 UI → [출결 시작] → 제한 시간 선택.
6. 4자리 코드·학생별 링크 생성됨.
7. 알림톡 또는 단체 채팅방으로 링크·코드 공유.
8. 화면에서 실시간 상태 모니터링.
9. 필요 시 수기 상태 변경.
10. [출결 종료] → 수업 기록에 결과 반영.
11. 알림톡 발송.

[시나리오 3: 학부모 링크 문제 처리]
1. 학부모에게 "링크 안 열린다"는 연락 받음.
2. /lesson → 해당 수업 → 해당 학생 재발송.
3. 학부모에게 새 문자의 링크를 사용하도록 안내.
4. 이전 링크는 자동 무효화됨.

[시나리오 4: 학생 출결 문제 처리]
1. 학생이 코드 입력했는데 실패했다는 연락.
2. 오류 종류 확인:
   - 코드 불일치 → 선생님 화면 4자리 대조.
   - 만료 → 수기 상태 변경.
   - 이미 완료 → 결과 화면 재확인 안내.
   - 세션 종료 → 수기 상태 변경.
3. 수기 수정: /attendance/sessions/:sessionId/students/:studentId PATCH.

[SECTION 12] 짧은 복붙용 FAQ
==============================================================

Q: 학부모 문자에 링크가 두 개예요.
A: 같은 수업에 재발송하면 이전 링크는 즉시 무효가 됩니다. 가장 마지막 문자의 링크만 열어 주세요.

Q: 출결을 같은 수업에서 두 번 할 수 있나요?
A: 설계상 수업당 한 번입니다. 재시작이 안 되면 수업 화면에서 출결을 수기로 수정해 주세요.

Q: AI 피드백이 안 나와요.
A: 학부모 페이지는 생성 중일 때 "준비 중" 문구만 보일 수 있습니다. 잠시 후(최대 약 3분) 다시 열어 보세요.
   선생님 화면의 학생 AI 분석은 AI 조교(/ai) 설정과 별개입니다.

Q: 점수 입력은 어떻게 하나요?
A: 85만 입력해도 되고, 85/100처럼 만점까지 같이 입력해도 됩니다.

Q: 왜 mock이라고 뜨나요?
A: 서버가 실제 카카오로 보내지 않는 연습 모드입니다.
   실서비스로 전환하려면 인프라 담당자에게 문의하세요.

Q: 출결 시작이 안 돼요.
A: 템플릿의 ATTENDANCE(출결) 항목이 정확히 1개인지 확인하세요.
   0개이거나 2개 이상이면 출결 세션 시작이 안 됩니다.

Q: 학부모 링크가 만료됐대요.
A: 해당 학생에게 다시 알림톡을 발송하면 새 링크가 생성됩니다.
   학부모에게 새로 받은 문자의 링크를 사용하도록 안내해 주세요.

Q: 엑셀에 학생이 두 번 나와요.
A: 정상입니다. 위쪽은 학부모 전화번호, 아래쪽은 학생 전화번호로 구분된 구조입니다.

Q: AI 조교 설정을 바꿨는데 학생 분석이 그대로예요.
A: 정상입니다. AI 조교 설정은 학부모 알림톡 피드백에만 적용되고,
   학생 대시보드 AI 분석은 별개 시스템입니다.

목적: 챗봇이 개발 문서 말투가 아니라, 선생님이 바로 이해하고 실행 가능한 문장으로 답하게 한다.

[SECTION 14] 출결 답변 표준 템플릿 (선생님용)
==============================================================

[기본형]
1. 출결 시작
- 수업 입력 화면에서 출결 시작을 누르면 4자리 코드와 마감 시간이 자동 생성됩니다.

2. 학생 체크
- 학생은 받은 링크에서 코드를 입력해 출결합니다.
- 결과는 출석/지각/결석으로 구분됩니다. (시스템 값: PRESENT/LATE/ABSENT)

3. 결과 확인/수정
- 선생님은 학생별 상태를 화면에서 바로 확인할 수 있고, 필요하면 수기로 수정할 수 있습니다.
- 수업 종료 후에는 출결 결과가 수업 기록에 반영됩니다.

4. 문제 발생 시
- 코드 오류: 코드 4자리 재확인
- 시간 초과: 마감 시간 확인
- 이미 체크됨: 중복 입력 불가(정상)


[지각 규칙 안내형]
- 기본적으로 시작 후 10분이 지나면 지각 처리됩니다.
- 단, 출결 제한 시간이 5분 이하인 경우에는 지각 없이 출석으로 처리됩니다.


[링크 만료 안내형]
- "학부모 링크는 재발송하면 이전 링크가 즉시 만료될 수 있습니다."
- "항상 가장 마지막에 받은 링크를 사용해 주세요."
